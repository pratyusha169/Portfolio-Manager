Executive Summary

This guide shows how to run multiple Claude agents in parallel using your existing Claude Max ($100/mo) and GitHub Copilot Pro subscriptions — no API keys required. You have four approaches: git worktrees (production-ready, independent sessions), agent teams (experimental, shared task list), Claude Code + Copilot side-by-side (privacy separation for PII data), and a combined strategy that assigns each task to the cheapest effective approach. From ~30 available plugins across Anthropic Official and community sources, we identified a Pareto-optimal set of 7 plugins that cover the broadest functionality with the least overlap and lowest token overhead. Install them in under 6 minutes with the copy-paste commands in Section 8. The guide also covers token cost optimization (use Sonnet for generation, Opus for reasoning) and strict rules about API key usage (extensions only — never import anthropic in your code).

2. The Four Parallel Approaches

A: Git Worktrees Production-Ready

Each worktree = independent branch + filesystem + Claude Code session.

# Terminal 1: Main development
claude

# Terminal 2: E2E testing (isolated branch)
claude --worktree e2e-testing

# Terminal 3: Feature work (isolated branch)
claude --worktree feature-x

How it works: claude --worktree <name> creates .claude/worktrees/<name>/ with a full repo copy on branch worktree-<name>. Each terminal runs an independent Claude session with its own context window.

Token tip: Use Sonnet (/model sonnet) for worktree sessions doing code generation. Reserve Opus for the main session.

B: Agent Teams Experimental

export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
claude

How it works: The lead creates child processes, each with its own context window. They share a task list (JSON) and send direct messages. The lead delegates and reviews.

Caveat: Split panes don't work in VSCode integrated terminal. Use external terminal + tmux.

Token cost: Linear with team size. 4-agent team = ~4x tokens.

C: Claude Code + GitHub Copilot Side-by-Side Production-Ready

Two separate VSCode windows, two separate extensions, two separate billing:

  • Window 1 (Claude Code): TaxUno Trade development
  • Window 2 (Copilot with Claude Opus): Schedule D Ledger (PII data scanning under Microsoft DPA)

Coordinated via the Mentu Ledger Agent Handoff System.

4. Deep Dive: Each Plugin Explained

4.1 feature-dev — 7-Phase Feature Development with Parallel Agents

What it does: Runs a structured workflow where each phase spawns parallel specialist agents:

Phase 1: UNDERSTAND  - Read request, ask clarifying questions
Phase 2: EXPLORE     - 2-3 parallel code-explorer agents scan codebase
Phase 3: PLAN        - Synthesize findings into feature plan
Phase 4: DESIGN      - 2-3 parallel architects propose approaches:
                        (a) Minimal  (b) Clean  (c) Pragmatic
Phase 5: CHECKPOINT  - YOU approve a design before any code
Phase 6: IMPLEMENT   - Code generation against approved design
Phase 7: REVIEW      - 3 parallel reviewers check result

Example:

/feature "Add carryover chain validation to the compute pipeline"

Token cost: ~15-25K per feature. Subagents use Sonnet by default (~5x cheaper than Opus).

When NOT to use: Simple bug fixes, one-line changes, docs updates. Just ask directly.

4.2 ralph-wiggum — Autonomous Iteration Loops

What it does: Creates a while (!done) loop for AI work. You define a task and completion condition. Claude runs, checks, retries until done.

How it works: A Stop hook re-injects your prompt after each response. Claude sees its previous output, identifies what's incomplete, continues.

/ralph-loop "Build pytest tests for carryover_service.py, 95% coverage.
When complete:
- All tests pass
- Coverage >= 95%
Output: <promise>TESTS_COMPLETE</promise>" \
  --completion-promise "TESTS_COMPLETE" \
  --max-iterations 20

Token cost: ~2-5K per iteration. Budget total = iterations x cost.

When NOT to use: Tasks needing human judgment mid-stream. Tasks where failure isn't detectable.

Key strength: Set it up, walk away. It iterates autonomously. Perfect for long-running operations.

4.3 code-review — 5 Parallel Sonnet Reviewers

What it does: Spawns 5 parallel Sonnet agents, each reviewing from a different angle: correctness, security, performance, style, test coverage.

Why Sonnet, not Opus: Five focused cheap reviews catch more than one expensive broad review. "Ensemble of cheap models beats one expensive model."

/code-review

Token cost: ~10-15K total across all 5 agents (Sonnet pricing).

When NOT to use: Tiny changes (1-2 files). Overhead isn't worth it for a typo fix.

4.5 spec-driven-workflow — Formal Requirements Before Code

What it does: Enforces "no code until spec is validated." Uses RFC 2119 (MUST/SHOULD/MAY), Given/When/Then acceptance criteria, edge case mapping.

1. ELICIT     - Gather requirements (use existing PRDs)
2. SPECIFY    - Write spec: FR/NFR/AC/Edge Cases
3. VALIDATE   - spec_validator.py scores 80+/100 to proceed
4. EXTRACT    - test_extractor.py auto-generates test cases
5. IMPLEMENT  - Code against spec (stops if ambiguous)
6. VERIFY     - Run extracted tests

Why it matters for TaxUno: Tax calculations are IRS-regulated. A spec with Given/When/Then test cases catches errors before code is written.

Token cost: Low. Spec is a markdown file. Validator is a Python script. Neither persists in context.

6. Token & Cost Optimization

The Three Rules

Rule 1: Sonnet for generation, Opus for reasoning

Opus: Architecture, code review, debugging

Sonnet: Code gen, tests, docs, batch refactors

Switch mid-session: /model sonnet

Rule 2: Be specific, not vague

Expensive: "Fix any issues in the codebase"

Cheap: "Fix sign convention in ledger_client.py line 98"

Rule 3: Use subagents to isolate context

Background subagents produce large output that gets summarized before returning to your main context. This saves tokens vs reading 20 files directly.

Cost Estimates (Claude Max)

WorkflowDaily CostModel Mix
Light (2-3 features)~$4-670% Sonnet, 30% Opus
Medium (parallel worktrees)~$8-1260% Sonnet, 40% Opus
Heavy (agent teams + loops)~$15-2050/50

Long-Running Operations

  • ralph-wiggum loops: Set up, walk away. Iterates until done or max iterations.
  • Background worktrees: claude --worktree long-task in a separate terminal.
  • Copilot window: Private data work on separate quota entirely.
